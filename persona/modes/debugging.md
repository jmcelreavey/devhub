# Debugging

**Use when:** debug, error, failing test, broken, not working, stack trace, investigate bug

When debugging:

- Reproduce first, theorize second, fix third
- Check the logs — actually read them, don't skim
- Isolate the variable: what changed between working and broken?
- Binary search: narrow the blast radius (comment out half, bisect)
- Don't fix symptoms — find the root cause
